Data Layer Integration
Hooks Used:
client/src/hooks/
├── useModuleAssessmentData.ts         // Fetches post-secondary cases
├── useModuleReportCase.ts             // Manages case selection
├── useMarkdownReport.ts               // Extracts markdown content
├── usePostSecondaryAssessmentData.ts  // Post-secondary specific data
├── usePostSecondaryAssessmentCases.ts // Case management
└── useReportDownloads.ts              // Export functionality
API Endpoints:
/api/assessment-cases-direct/post_secondary    // Fetches reports list
/api/analyze-assessment                        // Generates new reports
Database Table:
post_secondary_reports                         // Isolated from K-12 system
├── id (UUID primary key)
├── case_id (unique)
├── status ('pending'|'completed')
├── display_name
├── markdown_report                            // Main content
├── analysis_result (JSONB)
├── functional_barriers (JSONB)
└── accommodations (JSONB)

Navigation and URL Handling
URL Parameters Supported:
?case={caseId} - Auto-select specific case
&view=figma - Force enhanced view mode
&autoload=true - Enable auto-advancing sections
Demo URL format: /post-secondary-reports?case={caseId}&view=figma&autoload=true

Integration Steps for Another Agent
1. Copy Core Files:
# Main components
cp client/src/components/FigmaEnhancedReportViewer.tsx [target]/
cp client/src/components/PostSecondaryReportGenerator.tsx [target]/
# Supporting infrastructure
cp -r client/src/hooks/usePostSecondary* [target]/hooks/
cp -r client/src/hooks/useModuleReport* [target]/hooks/
2. Copy Assets:
cp attached_assets/icon*.png [target]/assets/
cp attached_assets/isotype*.png [target]/assets/  
cp attached_assets/image*.png [target]/assets/
3. Dependencies Required:
{
  "@radix-ui/react-*": "latest",
  "react-markdown": "^8.0.0",
  "remark-gfm": "^3.0.0",
  "lucide-react": "latest",
  "tailwindcss": "latest"
}
4. Vite Configuration:
Ensure vite.config.ts has asset alias:

resolve: {
  alias: {
    '@assets': path.resolve(__dirname, './attached_assets')
  }
}
5. Database Schema:
The agent will need to create the post_secondary_reports table using the schema from shared/schema.ts lines 107-121.

Key Architecture Decisions
Complete K-12 Isolation: Post-secondary data routes to separate post_secondary_reports table
Self-Contained UI: All styling and assets are internal, no external API dependencies
URL-Driven Navigation: Direct linkable sections for demo/sharing purposes
Mobile-First Design: Responsive across all device sizes
Brand Consistency: Official THRIVE color palette and iconography
This system is designed to be portable while maintaining full functionality and brand consistency.
